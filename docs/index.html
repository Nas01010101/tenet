Qwen Cloud Hackathon · Track 1 — Memory

Memory that knows when facts change.

Tenet is a bi-temporal memory system for LLM agents, built end-to-end on the Qwen stack. Every fact carries two clocks — when it became true, and when the agent learned it — so updates supersede instead of accumulate, and queries can time-travel: “where did I live in March?” gets March’s answer, not a lucky retrieval.

The idea

Two clocks per fact

Vector stores treat memory as a pile of text: every stale version of a fact stays retrievable forever, and the reader has to guess which one is current. Tenet stores facts, not chunks — each keyed, versioned, and stamped on two independent time axes:

key user::city
value "Berlin"
t_event 2025-05-05 ← when it became true in the world
t_ingest 2025-04-22 ← when the agent learned it
supersedes user::city = "Lisbon" (2025-01-10 → 2025-05-05)

That one structural decision buys three behaviors no flat store has: supersession (a new value closes the old one — exactly one current value per fact, no matter how many times it changed), time-travel (answer as-of any date on either axis), and principled forgetting (closed versions can be aged out without touching current knowledge).

Head-to-head

Same haystacks, same reader, same judge — vs ReMe

We ran Alibaba’s released memory framework ReMe (reme-ai 0.4.1.1) end-to-end on its own pipeline — auto_memory session notes → update_index → BM25 — against Tenet on LongMemEval_S (n=100, ~115k-token haystacks per question). Identical qwen3.7-plus reader and judge for every arm; we fixed four ReMe runtime defects to make the comparison fair, and documented each one.

QA accuracy, LongMemEval_S n=100

whiskers are Wilson 95% CIs · McNemar Tenet-vs-ReMe p≈2×10⁻⁶ · Tenet-vs-RAG +3pp (n.s.)

A supplementary arm running ReMe’s own answering agent as-shipped (vector+BM25 ReAct loop) scores 37.0% — statistically indistinguishable from its main-protocol 34.0% (p=0.55). The gap is ReMe’s result, not a protocol artifact: lossy note distillation plus keyword retrieval discards detail that plain chunk retrieval keeps. Tenet beats ReMe on every question type.

System

How it works — all Qwen, end to end

The shipped product runs entirely on Qwen Cloud: qwen3.6-flash distills, text-embedding-v4 embeds, qwen3.7-plus reads. A local-only mode (Ollama + bge-small + LoRA-distilled qwen2.5) reproduces the FactConsolidation headline at $0.

01 · DISTILL
Conversation → facts

Sessions are distilled into keyed, dated fact assertions — not chunks. A local LoRA distiller beats the cloud reference on key-consistency (0.775 vs 0.707).

02 · STORE
Bi-temporal board

Each assertion lands with (t_event, t_ingest). A new value for a key closes the old version — supersession is a first-class write, not a hope.

03 · RECALL
As-of retrieval

Queries resolve against the board as of any date on either clock; retrieval hands the reader 98.5% less context at 100% recall@10.

04 · READ
Answer + cite

A Chain-of-Note reader answers from the recalled facts — with a bounded repair path so long deliberation never truncates into garbage.

What we don’t claim

Multi-hop consolidation is honestly hard: 30.0 pooled SubEM (vs RAG’s 4.5) — better than the field, far from solved. Multi-session synthesis is a reported weak spot. Every negative result, ablation null, and run-invalidating bug we found is documented in the paper and benchmark log — the audit trail is part of the submission.
