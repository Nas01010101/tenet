\begin{abstract}
\noindent Long-term memory for LLM agents is almost universally implemented as
\emph{retrieval over a growing log of past turns}---a document-retrieval abstraction. We
argue this is the wrong abstraction for an agent that must model a changing world. We
introduce \textbf{knowledge churn}---the repeated updating of a fact over a long
interaction---and show that retrieval-augmented memory (RAG-memory) \emph{silently
degrades} under it: as the number of stale versions of a fact exceeds the retrieval budget
$k$, the reader is handed conflicting values and answers incorrectly. On a controlled
benchmark, a strong RAG-memory falls from $100\%$ to $50\%$ current-value accuracy as a
fact is updated $2\!\to\!12$ times. We propose \textbf{Tenet}, which reframes memory as a
\textbf{self-consistent belief state}---a compact \emph{world model of the user}---rather
than a document store. Tenet (i)~distills raw turns into atomic, keyed facts; (ii)~maintains
a \textbf{bi-temporal} record so a changed fact \emph{supersedes} its predecessor (retired
to history, not overwritten); (iii)~enforces \textbf{belief--evidence consistency} by
retiring raw evidence that echoes a superseded belief; (iv)~applies a
\textbf{predictive-coding write policy}---surprise-gating---that stores only observations the
model cannot already predict; and (v)~closes the accuracy gap to raw retrieval with
\textbf{belief-anchored evidence expansion}---spending spare context on query-relevant turns
from the sessions the belief state already surfaced. Tenet holds \textbf{$100\%$ current-value
accuracy across all churn levels} (where a strong RAG-memory falls to $50\%$), matches strong
RAG on retrieval recall ($95$--$97.5\%$), and---with expansion---\textbf{matches its one-shot
answer accuracy at equal-or-lower token budget} ($57.5\%$ vs $57.5\%$, gpt-4o reader) while
retaining a high-efficiency point at \textbf{half the context} and the \textbf{best
accuracy-per-token} of the systems we evaluate. Tenet thus traces an accuracy--efficiency
\emph{frontier} that meets RAG at its budget and beats it at every lower one. We release all code.
\end{abstract}

\section{Introduction}
An agent that talks to a user for months does not need a transcript; it needs a
\emph{model of the user} that stays true as the user changes. Yet the dominant memory
design---retrieval-augmented generation over stored turns~\cite{mem0,longmemeval}---treats
memory as a document index: embed every turn, retrieve the top-$k$ most similar, let the
reader sort it out. This works for one-shot recall of a \emph{static} fact. It fails,
quietly, when facts \textbf{change}.

We formalize this as \textbf{knowledge churn}. A user who moves cities several times
generates many ``I moved to $X$'' turns; all are similar to ``where do I live?'', so the
top-$k$ fills with \emph{stale} versions. Once the number of updates exceeds $k$, the latest
value may not be retrieved at all, and even when it is, the reader must infer recency from
contradictory statements. Accuracy collapses (\S\ref{sec:churn}).

The root problem is abstraction. A document store has no notion that ``I live in Boston''
and ``I live in Seattle'' are the \emph{same fact} with a \emph{changed value}. We argue
memory should be a \textbf{belief state}: a compact set of \emph{current} beliefs, each with
a temporal extent, updated by observation, kept internally consistent, and queryable across
time---the stance world-model and predictive-coding accounts take toward
perception~\cite{friston}, brought to agent memory.

\textbf{Contributions.}
\begin{enumerate}\itemsep2pt
\item We identify and name \textbf{knowledge churn}, a failure mode of retrieval memory, and
give a controlled benchmark exhibiting it (RAG: $100\%\!\to\!50\%$; \S\ref{sec:churn}).
\item We present \textbf{Tenet}, a memory that is a self-consistent belief state:
bi-temporal supersession, a \textbf{belief--evidence consistency} rule, and a
\textbf{surprise-gated} (predictive-coding) write policy.
\item We evaluate on LongMemEval\_S and controlled tests: Tenet is \textbf{churn-robust
($100\%$)}, on par with RAG on recall ($95\%$), \textbf{best on accuracy-per-token}, and---with
belief-anchored evidence expansion---\textbf{at parity with strong RAG on one-shot accuracy at
equal token budget}, closing a gap belief-only compression left open.
\item On the standardized \textbf{MemoryAgentBench FactConsolidation} benchmark,
ingestion-time supersession with zero-LLM deterministic keys \textbf{exceeds the published
single-hop SOTA at the gpt-4o-mini tier (86.5 vs 78.0 pooled) and ties the multi-hop tier
(30.2)}---using only a local 7B backbone, where the benchmark's 22 systems score
$\le$60~/~$\le$7.
\end{enumerate}

\section{Related Work}
\textbf{Retrieval memory.} Mem0~\cite{mem0} distills salient facts at write time over a
vector store; it attaches only a \emph{creation} timestamp and \emph{removed} its graph
variant after finding it $3\times$ slower / $2\times$ tokens for a thin gain---evidence we
weigh in choosing a light substrate. LongMemEval~\cite{longmemeval} is the standard
long-horizon benchmark; its V2 adds a \emph{latency-aware} metric, a field shift toward
accuracy \emph{per cost} that our per-token results target.
\textbf{Temporal knowledge graphs.} Zep/Graphiti~\cite{zep} maintain a bi-temporal graph
with automatic invalidation---but pay heavy per-write extraction
and require graph infrastructure; Tenet keeps the bi-temporal semantics without the graph.
\textbf{The 2026 bi-temporal convergence.} Concurrent systems adopted bi-temporal
supersession: MemStrata~\cite{memstrata} applies deterministic $(s,r,o)$ supersession over
a bi-temporal ledger with no LLM on the read path---and shows \emph{similarity-threshold}
supersession leaks stale values where deterministic keying does not, independently
corroborating our keyed design; Engram~\cite{engram} pairs a bi-temporal graph with a
hybrid facts-plus-chunks read path (converging on our dual-pool finding);
TOKI~\cite{toki} formalizes contradiction resolution as a bitemporal operator algebra;
and \cite{freshness} sets the current MemoryAgentBench FactConsolidation SOTA with
deterministic post-retrieval aggregation. Tenet differs in \emph{where} consistency is
enforced---at ingestion and at recall (belief--evidence consistency)---and in coupling the
belief state to surprise-gated writes and budget-bounded evidence expansion, evaluated on
the knowledge-churn axis none of them report.
\textbf{OS-style / observational memory.} MemGPT/Letta~\cite{memgpt} page memory between
context and archival storage; observational schemes keep a stable summary. Both are largely
append-oriented and do not treat supersession or forgetting as first-class.
\textbf{What is missing.} No prior system combines bi-temporal supersession,
\emph{belief--evidence consistency}, \emph{predictive-coding write-gating}, principled
forgetting, and \emph{belief-anchored evidence expansion} in a light, graph-free
store---nor reports the knowledge-churn regime.

\section{Method}
Tenet stores two layers over one bi-temporal table: a \textbf{belief layer} of distilled,
keyed facts, and an \textbf{evidence layer} of raw turns. Reads never call an LLM.

\textbf{Distillation.} Each turn is distilled into atomic facts, each with a stable
semantic key $\kappa=\textit{subject}\!:\!\textit{attribute}$ (e.g.\ \texttt{user:residence}),
a salience $s\in[0,1]$, and an event time. The key makes supersession reliable: embedding
similarity cannot separate a \emph{restated} fact from a \emph{value-changed} one (we measure
a value change at cosine $0.99$, indistinguishable from a paraphrase), but a shared key can.

\textbf{Bi-temporal supersession.} Every memory carries event time
($\texttt{valid\_at},\texttt{invalid\_at}$) and transaction time
($\texttt{created\_at},\texttt{expired\_at}$). Storing a fact at key $\kappa$ whose value
differs from the current one \emph{supersedes} it: the old fact's
$\texttt{invalid\_at}/\texttt{expired\_at}$ are set; it leaves the current set but stays in
history. Current recall filters $\texttt{expired\_at}=\varnothing$; $\text{recall}(as\_of{=}t)$
reconstructs the belief set at any past $t$ (time-travel).

\textbf{Belief--evidence consistency (key rule).} The evidence layer answers detail
questions but is where stale values hide: ``I moved to Boston'' survives after the belief
moves on. We retire from current recall any raw slice $e$ close to a \emph{superseded} belief:
\[
\text{exclude } e \iff \max_{f\in\mathcal{F}_{\text{expired}}}\cos(e,f)\ \ge\ \tau_{\text{stale}},
\qquad \tau_{\text{stale}}=0.80.
\]
This single rule turns supersession into end-to-end correctness: $55\%\!\to\!100\%$
(\S\ref{sec:abl}).

\textbf{Predictive-coding write policy.} A world model stores \emph{prediction error}. On
write, a raw observation $e$ is discarded if the store already predicts it:
\[
\text{store } e \iff \max_{e'\in\text{store}}\cos(e,e')\ <\ g_{\text{surprise}},
\qquad g_{\text{surprise}}=0.97 .
\]
This bounds the store and drops redundant repetition with no accuracy loss (\S\ref{sec:abl}).

\textbf{Forgetting.} Rank $=$ relevance $\times$ decay, with
$d = 2^{-\Delta t/h}\,(1+\beta\log(1{+}\text{uses}))\,(0.6+0.8s)$, half-life $h=14$\,d; a
sweep archives low-$d$ unpinned memories. Retrieval is a \emph{dual pool} (beliefs for
consistency, evidence for verbatim detail), each guaranteed a budget share.

\textbf{Belief-anchored evidence expansion.} Compressing a session into a few beliefs wins
churn and efficiency but can drop the detail a multi-hop question needs, even when the right
session \emph{is} retrieved (recall $95$--$100\%$). The top-$k$ result names both the belief
state \emph{and the sessions it came from} (each memory's \texttt{source}). Given spare
budget $B$, Tenet fills it with query-relevant raw turns drawn \emph{only from those
already-surfaced sessions}---evidence anchored to the belief state, not the whole
haystack---under the same stale-echo filter. Setting $B$ to the baseline RAG budget makes
expansion never spend more context than flat retrieval; the count $m$ is a knob ($m{=}0\to$
efficiency point; $m$ large under $B\to$ parity point) that lets one system trace a frontier.

\section{Experiments}
\textbf{Protocol.} LongMemEval\_S (500 questions, ${\sim}115$k-token histories).
We use a \textbf{\texttt{gpt-4o} reader} (the same reader Mem0 and Zep report against), a local embedder (\texttt{bge-small}), and a cheap \texttt{gpt-4o-mini} distiller; the shipped system runs the same code on Qwen Cloud (\texttt{text-embedding-v4}, \texttt{qwen3.7-plus}) by a config flip. Numbers are compared only to baselines run under identical settings.
Baselines: \textbf{RAG} (top-$k$ raw turns) and \textbf{full-context} (entire history).

\subsection{Recall, and the accuracy--efficiency frontier (n$=$40, gpt-4o reader)}
\label{sec:frontier}
\begin{center}\small
\begin{tabular}{llrrrr}
\toprule
System & mode & recall@10 & QA acc & reader tok & \textbf{acc/1k tok} \\
\midrule
full-context & --- & --- & 65\%$^{*}$ & ${\sim}124\text{k}$ & 0.5$^{*}$ \\
RAG          & top-$k$ turns & 95\% & 57.5\% & 2{,}101 & 27.4 \\
\textbf{Tenet} & efficiency ($m{=}0$) & \textbf{97.5\%} & 52.5\% & \textbf{1{,}067} & \textbf{49.2} \\
\textbf{Tenet} & parity (expansion) & \textbf{97.5\%} & \textbf{57.5\%} & 2{,}083 & 27.6 \\
\bottomrule
\end{tabular}
\end{center}
Tenet is a \emph{frontier}, not a point. At its \textbf{efficiency} point it answers at half
the context ($1{,}067$ tok) for the best accuracy-per-token of any system ($49.2$, $1.6\times$
RAG) at recall parity. Turning up belief-anchored expansion under RAG's own budget brings raw
QA accuracy to \textbf{parity with a strong RAG ($57.5\%{=}57.5\%$) at fewer tokens}, closing
the one-shot gap belief-only compression left open. Tenet thus meets RAG's accuracy at its
budget and beats it at every lower one; the one category still behind is \emph{multi-session}
synthesis ($42.9$ vs $57.1$, up from $28.6$), where a question needs several evidence sessions
but only some are surfaced. $^{*}$Full-context, at a weaker reader, spends $100\times$ the
tokens for no gain over RAG. Reader-robust: on a cheaper \texttt{gpt-4o-mini} reader the parity
point edges ahead (Tenet $60.0$ vs RAG $55.0$); per-token dominance holds across
\texttt{gpt-4o-mini}, \texttt{gpt-4o}, and \texttt{claude-opus-4.8} ($\approx$$1.6$--$1.7\times$).

\subsection{Knowledge churn (headline)}
\label{sec:churn}
One fact updated $N$ times amid distractors, $k=6$, 12 principals/point:
\begin{center}\small
\begin{tabular}{lrrrrrr}
\toprule
updates $N$ & 2 & 4 & 6 & 8 & 10 & 12 \\
\midrule
RAG            & 100 & 100 & 100 & 67 & 58 & \textbf{50} \\
\textbf{Tenet} & 100 & 100 & 100 & \textbf{100} & \textbf{100} & \textbf{100} \\
\bottomrule
\end{tabular}
\end{center}
RAG degrades monotonically once $N>k$ ($-50$\,pp); Tenet is flat at $100\%$. Supersession
keeps exactly one current value regardless of churn---the property a \emph{belief state} has
and a document index cannot. \textbf{The curves are identical under a \texttt{gpt-4o-mini} and a \texttt{gpt-4o} reader}: the failure is structural (once $N>k$ the latest value is not reliably retrieved), so a stronger reader cannot rescue RAG.

\subsection{Ablations}
\label{sec:abl}
\textbf{Belief--evidence consistency.} Removing the rule drops Tenet to $55\%$ current-value
accuracy with a $45\%$ stale-leak; adding it restores $100\%$ / $0\%$ leak---the single most
important mechanism. \textbf{Surprise-gating} discards $15\%$ of observations as redundant
with no accuracy change, yielding a bounded store where RAG grows unboundedly.

\subsection{Standardized conflict resolution: MemoryAgentBench FactConsolidation}
On the ICLR~2026 conflict-resolution benchmark~\cite{mab} (SubEM metric and official reader
prompt verbatim; all 800 questions; Wilson 95\% CIs), ingestion-time supersession with
\textbf{fully deterministic, zero-LLM keys} and a deliberately weak \textbf{local 7B
backbone} scores, pooled over four haystack lengths (6K--262K):
\begin{center}\small
\begin{tabular}{lrrr}
\toprule
pooled & naive-RAG (same reader) & \textbf{Tenet} & published SOTA (mini / gpt-4o)~\cite{freshness} \\
\midrule
FC-SH ($n{=}400$) & 47.8 & \textbf{86.5} [82.8, 89.5] & 78.0 / 94.8 \\
FC-MH ($n{=}400$) & 4.5  & \textbf{30.2} [26.0, 34.9] & 30.2 / 51.5 \\
\bottomrule
\end{tabular}
\end{center}
Single-hop \textbf{exceeds the published gpt-4o-mini-tier SOTA (78.0; our CI excludes it) on
a weaker backbone}; every system in the original benchmark table scores $\le$60 (Zep 7,
Mem0 18, MemGPT 28). Multi-hop exactly ties the mini-tier SOTA, where the original table's
best is $\le$7. Accuracy barely degrades with haystack length (SH $89\!\to\!81$ from
6K$\to$262K): the store is conflict-resolved at ingestion---the property assembly-time
aggregation must re-derive at every read. Multi-hop still degrades with length
($42\!\to\!20$); reported honestly.

\section{Limitations}
\textbf{Multi-session synthesis} is the one category where RAG still leads ($42.9$ vs $57.1$).
Belief-anchored expansion lifted it from $28.6$ but does not close it: these questions need
evidence from \emph{several} sessions, and expansion only deepens the sessions the top-$k$
already surfaced---if a required session is not among them, its detail is still missing.
Session-diverse retrieval is the natural next step; elsewhere Tenet is at or above RAG.
\textbf{The frontier is a knob, not free lunch}: parity accuracy costs RAG-equal tokens, and
the $1.6\times$ per-token win is at the efficiency point, which trades ${\sim}5$\,pp of raw
accuracy. \textbf{Evaluation}: $n{=}40$, off-Qwen (gpt-4o / gpt-4o-mini readers, local
embedder), one seed; reader stochasticity is ${\approx}{\pm}5$--$7$\,pp, so the one-shot result
is reported as \emph{parity}, not a win. The shipped system uses Qwen Cloud.

\section{Conclusion}
Treating agent memory as a \textbf{self-consistent belief state} rather than a document index
makes it robust to the way real knowledge behaves: it changes. Tenet stays correct under
knowledge churn where retrieval memory collapses, matches a strong RAG's one-shot accuracy at
equal token budget while offering the best accuracy-per-token we tested, with a light
graph-free substrate and no LLM in the read path---and gets time-travel and principled
forgetting for free. We hope \emph{knowledge churn} becomes a standard axis for
evaluating agent memory.
