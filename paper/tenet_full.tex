\begin{abstract}
\noindent Long-term memory for LLM agents is almost universally implemented as
\emph{retrieval over a growing log of past turns}. We argue this is the wrong abstraction
for an agent that must model a \emph{changing} world, and we characterize the failure as
\textbf{knowledge churn}: as a fact is updated $N$ times over a long interaction, stale
versions crowd the retrieval budget $k$, and once $N>k$ the current value is no longer
reliably retrieved. On a controlled benchmark a strong RAG memory falls from $100\%$ to
$50\%$ current-value accuracy as a fact is updated $2\!\to\!12$ times---under both a
\texttt{gpt-4o-mini} and a \texttt{gpt-4o} reader, so the collapse is structural, not a
reader artifact. We propose \tenet{}, which reframes memory as a \textbf{self-consistent
belief state}: (i) turns are distilled into atomic facts with stable semantic keys;
(ii) a \textbf{bi-temporal} record lets a changed fact \emph{supersede} its predecessor at
ingestion (history kept, time-travel queryable); (iii) a \textbf{belief--evidence
consistency rule} retires raw evidence that echoes a superseded belief; (iv) a
\textbf{surprise-gated write policy} stores only observations the current state does not
already predict; and (v) \textbf{belief-anchored evidence expansion} spends spare context
on raw turns from the sessions the belief state already surfaced. The read path never
calls an LLM. \tenet{} holds $100\%$ under churn at every update level; on
MemoryAgentBench \textsc{FactConsolidation} (ICLR~2026) it scores $86.5\%$ single-hop
\ci{82.8}{89.5}---above the published \texttt{gpt-4o-mini}-tier state of the art
($78.0$) despite using a local 7B backbone---and ties the multi-hop state of the art
($30.2$); on MemoryAgentBench \textsc{Accurate Retrieval} it averages $59.3$, second only
to HippoRAG-v2 ($65.1$) among published systems while being the only one whose ingestion
never calls an LLM, and beats the field on EventQA ($70.7$ vs $67.6$; Wilson CI excludes).
All numbers are reproducible from released code with official metrics and prompts verbatim.
\end{abstract}

\section{Introduction}
An agent that talks to a user for months does not need a transcript; it needs a
\emph{model of the user} that stays true as the user changes. Yet the dominant memory
design---retrieval-augmented generation over stored turns~\cite{mem0,longmemeval}---treats
memory as a document index: embed every turn, retrieve the top-$k$ most similar at query
time, and let the reader sort it out. This works for one-shot recall of a \emph{static}
fact. It fails, quietly, when facts \emph{change}.

We formalize this failure as \textbf{knowledge churn}. A user who moves cities several
times generates many ``I moved to $X$'' turns; all are similar to the query ``where do I
live?'', so the top-$k$ fills with \emph{stale} versions. Once the number of updates $N$
exceeds the budget $k$, the latest value may not be retrieved at all; even when it is, the
reader must infer recency from a pile of contradictory statements. Accuracy collapses
(\S\ref{sec:churn}), and---critically---a stronger reader cannot rescue it, because the
failure occurs before the reader sees anything.

The root problem is the abstraction. A document store has no notion that ``I live in
Boston'' and ``I live in Seattle'' are the \emph{same fact} with a \emph{changed value};
both are just passages. We argue memory should instead be a \textbf{belief state}: a
compact set of \emph{current} beliefs about the world, each with a temporal extent,
updated by observation, kept internally consistent, and queryable across time. This is
the stance world-model and predictive-coding accounts take toward
perception~\cite{friston}; we bring it to agent memory and pay particular attention to
\emph{where} consistency is enforced: at ingestion, so the store never contains a stale
current value, rather than at assembly time, where every read must re-derive it.

\paragraph{Contributions.}
\begin{enumerate}
\item We identify and name \textbf{knowledge churn}, a structural failure mode of
  retrieval-augmented memory, give a controlled benchmark exhibiting it (RAG:
  $100\%\!\to\!50\%$ as a fact is updated $2\!\to\!12\times$; reader-invariant), and show
  a belief-state memory is immune by construction (\S\ref{sec:churn}).
\item We present \tenet{}: bi-temporal keyed supersession at ingestion, a
  \textbf{belief--evidence consistency rule} that retires stale raw evidence, a
  \textbf{surprise-gated write policy}, salience-decay forgetting, and budget-bounded
  \textbf{belief-anchored evidence expansion}---in a light, graph-free store whose read
  path never calls an LLM (\S\ref{sec:method}).
\item On the standardized \textbf{MemoryAgentBench}~\cite{mab} we evaluate both core
  competencies with official metrics and prompts verbatim, Wilson CIs, and matched-reader
  controls. \textsc{FactConsolidation}: single-hop $86.5$ \ci{82.8}{89.5}, above the
  published mini-tier SOTA ($78.0$~\cite{freshness}, CI excludes) on a \emph{weaker}
  (local 7B) backbone; multi-hop $30.2$, tying the same SOTA, where every system in the
  original benchmark table scores $\le 7$. \textsc{Accurate Retrieval}: average $59.3$,
  second among published systems to HippoRAG-v2's $65.1$~\cite{hipporag2}---which runs
  LLM OpenIE over every context token, where \tenet{} ingests with embeddings
  only---and ahead of the field on EventQA ($70.7$ vs $67.6$, CI excludes)
  (\S\ref{sec:mab}).
\item We reimplement four published memory methods (Mem0-style consolidation, CAR
  aggregation~\cite{freshness}, a HippoRAG-v2-style OpenIE+PPR graph, and
  MemAgent-style~\cite{memagent} overwrite memory) as arms in the \emph{same} harness with
  the \emph{same} backbone, isolating the memory mechanism as the only variable
  (\S\ref{sec:baselines}).
\end{enumerate}

\section{Related work}\label{sec:related}
\textbf{Retrieval memory.} Mem0~\cite{mem0} distills salient facts at write time over a
vector store with entity links; it attaches only a \emph{creation} timestamp and notably
removed its graph variant after finding it $3\times$ slower for a thin gain---evidence we
take seriously in choosing a light vector substrate. LongMemEval~\cite{longmemeval} is the
standard long-horizon benchmark; its V2~\cite{lmev2} adds a latency-aware metric,
signalling a field shift toward accuracy \emph{per cost}, which our per-token results
target.

\textbf{Temporal knowledge graphs.} Zep/Graphiti~\cite{zep} maintains a \emph{bi-temporal}
knowledge graph (valid + transaction time) with edge invalidation---the closest prior
system in temporal semantics. It differs in substrate and mechanism: invalidation is
LLM-mediated at graph-construction time over (subject, predicate, object) edges, reads
traverse a graph, and neither a belief--evidence consistency rule over raw text nor
surprise gating is present. \tenet{} keeps the bi-temporal semantics without the graph,
and its supersession is deterministic given the distilled key.

\textbf{The 2026 bi-temporal convergence.} Concurrently with this work, several systems
adopted bi-temporal supersession: MemStrata~\cite{memstrata} applies a deterministic
(subject, relation, object) supersession rule over a bi-temporal ledger with no LLM in
the read path---and shows \emph{similarity-threshold} supersession leaks stale values
where deterministic keying does not, independently corroborating our keyed design;
Engram~\cite{engram} pairs a bi-temporal knowledge graph with a hybrid
facts-plus-raw-chunks read path (converging on our dual-pool finding);
TOKI~\cite{toki} gives contradiction resolution a formal bitemporal operator algebra. On
conflict-resolution benchmarks, \cite{freshness} shows the \emph{assembly} step dominates:
deterministic $\max(\text{serial})$ aggregation after retrieval sets the published SOTA on
MemoryAgentBench FactConsolidation. \tenet{} differs from all of these in \emph{where}
consistency is enforced---at ingestion (the store never contains a stale current value)
\emph{and} at recall (belief--evidence consistency retires stale raw evidence)---and in
coupling the belief state to a write policy (surprise gating) and a budget-bounded
evidence expansion, evaluated on the knowledge-churn axis none of them report.

\textbf{Memory in the Qwen ecosystem.} Alibaba's 2026 memory line takes the opposite
design point: QwenLong-L1.5~\cite{qwenlong} reaches $76.4$ on LongMemEval by RL-training a
30B reasoner that reads up to 128K tokens per query; AgeMem~\cite{agemem} RL-trains the
memory policy itself; ActMem~\cite{actmem} (Alibaba-co-authored) reaches $75.6$ with
LLM-built causal graphs at ingestion. All three spend heavily---at training time,
ingestion time, or read time. \tenet{} reaches $60.0$ on the same benchmark at the same
reader tier with zero-LLM ingestion and $\sim$2K read tokens per query ($\approx 79\%$ of
QwenLong-L1.5's score at under $2\%$ of its read budget), and its structural wins
(churn, conflict resolution) survive on a 7B backbone.

\textbf{Active memory navigation.} A concurrent 2026 line treats memory access itself as
an agentic, learned process. NapMem~\cite{napmem} exposes a four-level memory pyramid
(raw conversation / typed records / topic tracks / user profile) as a structured action
space and trains a GRPO policy that descends the pyramid and stops once evidence is
sufficient---a trained 9B policy beats an untrained 397B model on memory benchmarks.
QwenLong-L1.5~\cite{qwenlong} additionally plans a navigational path over memory chunks,
lifting LongMemEval by $+15.6$ points. MemFlow~\cite{memflow} is training-free: an LLM
router selects a memory tier per query, and an LLM Validator retries with a heavier tier
on a low-confidence read. MemReader~\cite{memreader} moves the same active-navigation
idea to the write side, extracting memory rather than passively logging it. AgeMem
\cite{agemem} and Mem-$\alpha$~\cite{memalpha} RL-train the read/construction policy
itself; MemCog~\cite{memcog} reports a similar cognitive-navigation frame (affiliation
unverified). All of these make the \emph{read} path agentic: an LLM or an RL policy
decides how deep to go and when to stop. \tenet{} takes the opposite bet on the read
side---it keeps recall LLM-free and gets adaptive depth from an embedding saturation
gate (\S\ref{sec:navigate}) instead of a learned or LLM-mediated stop.

\textbf{OS-style and agentic memory.} MemGPT/Letta~\cite{memgpt} pages memory between a
context ``RAM'' and archival ``disk''; A-Mem~\cite{amem} lets the agent link and evolve
notes. Both are largely append-oriented: fact supersession, consistency between distilled
and raw layers, and forgetting are not first-class operations.

\textbf{What is missing.} No prior system combines (a) ingestion-time bi-temporal keyed
supersession, (b) an explicit belief--evidence consistency rule spanning the distilled and
raw layers, (c) surprise-gated writes, (d) principled forgetting, and (e) belief-anchored
budget-bounded expansion in a graph-free store with an LLM-free read path---nor does any
report the knowledge-churn regime.

\section{Knowledge churn}\label{sec:problem}
Let a fact with key $\kappa$ take values $v_1,\dots,v_N$ over an interaction, each update
producing at least one stored turn. A flat retriever with budget $k$ ranks turns by
query similarity; all $N$ versions are near-equally similar to a query about $\kappa$
(paraphrases of the same statement), so the expected number of \emph{stale} passages in
the top-$k$ grows with $N$, and for $N>k$ the \emph{current} value $v_N$ competes with
$N-1$ distractors for $k$ slots with no ranking signal that prefers it. Two consequences:
(1) retrieval of $v_N$ becomes unreliable; (2) even when retrieved, the reader receives
contradictory values and must infer recency from text alone. Both are properties of the
\emph{store}, not the reader---which predicts the reader-invariance we observe in
\S\ref{sec:churn}. A belief state removes the problem at the source: the store contains
exactly one current value per key, so retrieval of the current value is $O(1)$ in $N$.

\section{Method}\label{sec:method}
\tenet{} stores two layers over one bi-temporal table: a \textbf{belief layer}
$\mathcal{B}$ of distilled, keyed facts and an \textbf{evidence layer} $\mathcal{E}$ of
raw turns. Writes maintain consistency; reads are deterministic lookups plus embedding
search---no LLM call.

\subsection{Distillation into keyed beliefs}
Each turn is distilled by a small LLM into atomic facts, each with a stable semantic key
$\kappa=\textit{subject}{::}\textit{attribute}$ (e.g.\ \texttt{user::residence}), a
salience $s\in[0,1]$, and an event time. The key is what makes supersession reliable:
embedding similarity cannot separate a \emph{restated} fact from a \emph{value-changed}
one (we measure a residence value change at cosine $0.99$, indistinguishable from a
paraphrase), but a shared key can. Where the source text is already structured (as in
benchmark fact streams), keys are computed deterministically from the text with no LLM at
all (\S\ref{sec:fc}).

\subsection{Bi-temporal supersession}
Every memory carries event time (\texttt{valid\_at}, \texttt{invalid\_at}) and transaction
time (\texttt{created\_at}, \texttt{expired\_at}). Storing a fact with key $\kappa$ whose
value differs from the current fact at $\kappa$ \emph{supersedes} it: the old fact's
\texttt{invalid\_at}/\texttt{expired\_at} are set; it leaves the current set but remains
in history. Current recall filters \texttt{expired\_at IS NULL};
$\textit{recall}(\textit{as\_of}=t)$ reconstructs the belief set as of any past $t$
(time-travel). Supersession is thus enforced \emph{at ingestion}: the current store never
contains a stale value, and no per-read aggregation is needed to re-derive freshness.

\subsection{Belief--evidence consistency}\label{sec:bec}
The evidence layer lets the reader answer detail questions, but it is also where stale
values hide: a raw turn ``I moved to Boston'' survives even after the belief
\texttt{user::residence} moves on. We therefore retire from current recall any raw slice
$e$ whose embedding is close to a \emph{superseded} belief:
\begin{equation}
\text{exclude } e \;\Leftrightarrow\; \max_{f\in\mathcal{B}_{\text{expired}}}
\cos\!\big(\phi(e),\phi(f)\big) \;\ge\; \tau_{\text{stale}}, \qquad
\tau_{\text{stale}}=0.80 .
\end{equation}
This single rule turns supersession from a fact-layer nicety into end-to-end correctness:
it takes current-value accuracy from $55\%$ to $100\%$ (\S\ref{sec:ablation}).

\subsection{Surprise-gated writes (predictive coding)}
A world model stores \emph{prediction error}, not everything. On write, a raw observation
$e$ is discarded if the store already predicts it, i.e.\ it is near-identical to an
existing slice:
\begin{equation}
\text{store } e \;\Leftrightarrow\; \max_{e'\in\mathcal{E}}
\cos\!\big(\phi(e),\phi(e')\big) \;<\; g_{\text{surprise}}, \qquad
g_{\text{surprise}}=0.97 .
\end{equation}
This bounds the store and drops redundant repetition with no accuracy loss
(\S\ref{sec:ablation}).

\subsection{Forgetting and dual-pool recall}
Each memory's rank is relevance times a decay factor
$d = 2^{-\Delta t/h}\cdot(1+\beta\log(1+\textit{uses}))\cdot(0.6+0.8s)$ with half-life
$h=14$\,d; a sweep archives current, unpinned memories with $d$ below a threshold; pinned
identity facts never decay. Retrieval is a \textbf{dual pool}---beliefs for consistency,
evidence for verbatim detail---guaranteeing each a share of the budget.

\subsection{Fact dynamics: learning how facts change}\label{sec:dynamics}
A belief state becomes a world model when it can predict its own drift. The
bi-temporal ledger is a training set for exactly this: every superseded fact is an
observed lifetime ($\texttt{valid\_at}\!\to\!\texttt{invalid\_at}$) and every current
fact a right-censored one. Per key \emph{class} $c$ (the attribute part of
$\kappa$), we place a conjugate Gamma prior on an exponential change rate; with
$n_c$ observed supersessions and total exposure $T_c$, the posterior-predictive
survival is Lomax:
\begin{equation}
P(\text{still valid after } \Delta t \mid c) \;=\;
\Big(\tfrac{\beta_0 + T_c}{\beta_0 + T_c + \Delta t}\Big)^{\alpha_0 + n_c},
\end{equation}
so ``residence'' learns a slow hazard and ``mood'' a fast one, per user, in closed
form, with no LLM. A second ledger statistic captures \emph{ripple}: the smoothed
co-supersession probability $P(c' \text{ changes} \mid c \text{ changed})$; when a
correlated key changes, the survival of its neighbors is discounted
($S \to S^{1+\gamma\,\text{bump}}$), so one observation updates beliefs about
unobserved attributes---the defining world-model behavior. Both surface as a
\emph{confidence} on every recalled fact and an $\textit{uncertain\_facts}()$ list
the agent can use to re-verify with the user. Deliberately, confidence does
\textbf{not} demote a fact's rank: a doubted fact is still the best known answer,
and we measured rank-demotion re-creating the churn failure of \S\ref{sec:churn}
(current value pushed behind distractors) before reverting to annotation-only.

\paragraph{A trained neural dynamics module.} The closed-form model is memoryless by
family; we also train a compact (276k-param, 1-layer GRU) \emph{neural temporal point
process} over fact-update events, whose Weibull hazard head admits non-constant
per-key hazards and whose auxiliary heads predict the next key to change (learned
ripple) and the next value (contrastively, over embeddings). On a controlled synthetic
corpus with planted periodic, bursty, and cascading structure---provably outside the
closed-form family---the neural model reduces held-out NLL from 2.76 to 1.99
nats/event (paired-bootstrap gain $0.81$, 95\% CI $[0.70, 0.93]$), halves survival
mis-calibration (KS $0.56\!\to\!0.26$), and predicts the next value at $0.997$
recall@5 vs $0.05$ chance, stable over five seeds. On real counterfactual-update
chains the result is deliberately mixed: better calibration and observed-lifetime
likelihood on FactConsolidation, but the censoring-dominated aggregate NLL loses to a
marginal-rate baseline, and LongMemEval yields too few clean chains ($n{=}21$) to
conclude. The exported model is a 1\,MB numpy artifact evaluated in $106\,\mu$s,
opt-in behind an env flag with default behavior unchanged, and passes the
churn-integrity guard.

\subsection{Belief-anchored evidence expansion}\label{sec:expansion}
Compressing a session into a few keyed beliefs wins churn and efficiency but can drop the
fine detail a multi-hop question needs, even when the right session \emph{is} retrieved.
We recover it without reverting to flat retrieval: the top-$k$ dual-pool result names both
the belief state and the sessions it came from. Given spare context budget $B$, \tenet{}
fills it with additional query-relevant raw turns drawn \emph{only from those
already-surfaced sessions}, subject to the same consistency filter (\S\ref{sec:bec}) so no
stale value re-enters. $B$ is set to the baseline RAG budget, so expansion never spends
more context than flat retrieval. The knob traces an accuracy--efficiency frontier: $m=0$
is the efficiency point; $m$ large under budget $B$ is the parity point.

\subsection{Saturation-gated navigation}\label{sec:navigate}
Belief-anchored expansion (\S\ref{sec:expansion}) draws raw evidence from a session
boundary; associative multi-hop recall goes further by re-conditioning the cue on
retrieved evidence and re-scoring the store, but at a caller-fixed depth
$\textit{hops}=N$. A fixed $N$ over-fetches simple queries (the extra hops add
distractors that hurt the reader) and under-fetches genuinely multi-hop ones.
\texttt{navigate()} replaces the fixed depth with an iterative, budget-bounded
hop-deepening loop that reuses belief-anchored expansion at every hop and adopts hop
$d{+}1$ only while its newly-reached evidence is still informative:
\begin{equation}
\text{adopt hop } d{+}1 \;\Longleftrightarrow\;
\max_{e\,\in\,\textit{new}_{d+1}} \textit{score}(e) \;\ge\; \tau_{\text{gain}},
\qquad \tau_{\text{gain}} = 0.15,
\end{equation}
where $\textit{new}_{d+1}$ is the evidence reached at depth $d{+}1$ but not at depth
$d$. When no hop clears the gate, or the hop budget ($\le 4$, mirroring NapMem's
tool-call budget) is exhausted, navigation stops and returns the last adopted pool,
dropping the marginal hop's distractors rather than keeping them. The gate is an
embedding-only marginal-relevance test over scores \tenet{} already computes for
recall, so it adds no LLM call and requires no training: it is the training-free
counterpart to NapMem's learned stop and to MemFlow's LLM-Validator retry, running at
embedding-search latency ($\sim$ms) instead of an LLM round trip.

We validate the mechanism, not a benchmark number. On a deterministic, torch-free
synthetic store (hashed bag-of-words embeddings, no network call), \texttt{navigate()}
(a) reaches a bridge fact invisible to broad top-$k$ retrieval via an associative hop,
and (b) stops early (hop 2) on a saturated single-hop query while continuing to hop 3
on a genuinely multi-hop one, confirming the gate behaves as intended (multi-hop reach,
single-hop early-stop). A subsequent seeded A/B on \textsc{FactConsolidation} 6k cells
at the \texttt{qwen3.7-plus} tier ($n{=}20$ per cell, same reader, prompts, and seed;
retrieval-pool construction the only variable) measured \emph{no benefit}: accuracy
identical to default recall on both cells, and $-5$ to $-10$pp against a fixed-4-hop
arm, all within Wilson CIs. Dumped trajectories confirm the mechanism executed as
designed (pools differed from baseline in $21/21$ rows, adaptive depth $2$--$3$): a
sufficiently strong reader is robust to the larger fixed pool, so adaptive
early-stopping buys no answer precision at this tier. We therefore report navigation
as a mechanism contribution only, with a measured null at the tested tier.

\section{Experimental setup}\label{sec:setup}
\textbf{Benchmarks.} (1) A controlled \textbf{knowledge-churn} benchmark (one fact updated
$N$ times amid distractors); (2) a controlled \textbf{knowledge-update} set for ablations;
(3) \textbf{LongMemEval\_S}~\cite{longmemeval} ($\sim$115K-token histories); (4--5) both
core competencies of \textbf{MemoryAgentBench}~\cite{mab} (ICLR 2026):
\textsc{FactConsolidation} (conflict resolution; all 800 questions) and
\textsc{Accurate Retrieval} ($\sim$2{,}000 questions over 22 contexts of 197K--534K
tokens).

\textbf{Protocol fidelity.} For MemoryAgentBench we use the official metrics and prompts
\emph{verbatim} from the benchmark code: SubEM for FactConsolidation and RULER-QA;
multiple-choice accuracy for EventQA; the official LLM-judge templates for
LongMemEval(S*). Readers are matched to the published tier being compared
(\texttt{gpt-4o-mini} for AR; a deliberately \emph{weaker} local \texttt{qwen2.5:7b} for
FC). Every \tenet{} number is paired with a naive-RAG control sharing the reader,
chunking, and embedder, so the memory mechanism is the only variable.

\textbf{Integrity rules.} API failures are excluded, never scored as wrong (0 exclusions
in FC; exclusions reported where they occur); abstentions are scored wrong; ingestion
caches are keyed by the hash of the exact chunk list; we report Wilson 95\% CIs and dump
every miss for error analysis. All numbers reproduce from single documented commands.

\subsection{Ablations}\label{sec:ablation}
On the controlled knowledge-update set, removing the belief--evidence consistency rule
(\S\ref{sec:bec}) drops current-value accuracy from $100\%$ to $55\%$ with a $45\%$
stale-leak rate; restoring it recovers $100\%$/0\%. It is the single most important
mechanism. Surprise gating discards $15\%$ of observations as redundant with no accuracy
change, bounding the store where RAG grows without bound.

\subsection{Case study: web-agent trajectory memory (LongMemEval-V2)}\label{sec:lmev2}
LongMemEval-V2~\cite{lmev2} shifts the setting from chat to \emph{web-agent trajectory}
histories (up to 115M tokens; DOM states, actions, URLs) and scores a latency--accuracy
frontier (LAFS) against a strong RAG reference (51.0\% accuracy) using a mandated
Qwen3.5-9B reader. We adapted \tenet{}'s ingestion to this regime as a stress test of
the retrieval substrate. A naive port (bge-small dense over trimmed DOM slices)
retrieved the gold evidence for only $\sim$12\% of the 295 deterministic-metric
questions. Three LLM-free changes rebuilt it: structure-aware chunking that keeps
(action, URL, salient DOM) units together and dedups cross-state DOM; query cleaning;
and hybrid retrieval---Qwen3-Embedding dense fused with BM25 by reciprocal-rank fusion.
Gold-evidence recall rose to \textbf{59.7\%} at a 48K reader budget (63.3\% at 72K)
against a measured corpus ceiling of $\sim$92\%, with query latency $\sim$0.25\,s---the
fast corner of the LAFS frontier. End-to-end accuracy proved \emph{reader-gated}, not
retrieval-gated---and the gate is the \emph{reasoning mode}, not precision: a 4-bit
local reader scores $45.1\%$ \ci{39.5}{50.8}, an 8-bit one $40.3\%$, and the
full-precision hosted reader \emph{without} extended thinking $40.7\%$ (it
over-abstains), all pinned near the same
$P(\text{correct}\,|\,\text{gold retrieved})\approx0.5$--$0.6$ extraction ceiling.
The leaderboard's default---extended thinking, $\sim$8K reasoning tokens per
question---is what converts retrieval headroom to accuracy, and lies outside our
compute budget. The
case study supports the paper's central claim in a second modality: the memory
substrate---LLM-free ingestion and sub-second reads---transfers, and where the system
falls short the binding constraint is measurable and external to the memory.

\section{Discussion}
\textbf{Where the belief state wins.} Both MAB competencies reward what supersession
provides: FactConsolidation directly (the store contains exactly one current value; no
per-read freshness inference), EventQA indirectly (date-aware structured chunks preserve
event order that flat chunking destroys). The churn benchmark isolates the mechanism:
immunity is by construction, not tuning.

\textbf{Where it loses.} Multi-hop chaining over narrative text (RULER MH-QA) rewards
explicit graph traversal; a keyed belief state has no edges to walk. Multi-session
synthesis needs evidence from several sessions at once; expansion only deepens sessions
already surfaced. Both are retrieval-topology limits, not consistency limits---graph-free
was a deliberate trade (Mem0's own ablation found graphs $3\times$ slower for thin gains),
and the gap quantifies what that trade costs.

\textbf{Cost asymmetry.} Among published systems above 50 on AR, \tenet{} is the only one
with LLM-free ingestion \emph{and} LLM-free reads; HippoRAG-v2 spends LLM calls per
ingested token, QwenLong-L1.5 spends 128K read tokens per query, ActMem builds LLM causal
graphs at ingestion. At equal accuracy this asymmetry compounds over an agent's lifetime:
ingestion happens once per fact, reads happen forever, and \tenet{} prices both at
embedding cost.

\section{Limitations}\label{sec:limits}
\textbf{Multi-session synthesis} still trails RAG (42.9 vs 57.1); session-diverse
retrieval is the natural next step. \textbf{Multi-hop chaining} trails graph memory (45
vs 66). \textbf{The frontier is a knob, not a free lunch}: parity accuracy costs
RAG-equal tokens; the $1.6\times$ per-token win trades $\sim$5\,pp of one-shot accuracy.
\textbf{Evaluation scale}: LongMemEval numbers are $n{=}40$, one seed, off-Qwen readers
(reader noise $\approx\pm5$--7\,pp; hence ``parity'', not a win); MAB numbers are
full-scale (800 and $\sim$2{,}000 questions) with CIs. \textbf{Distillation quality}: in
free-form conversation, keys come from a small LLM distiller; a missed key match degrades
supersession to ordinary storage (fail-soft, but the churn guarantee is only as good as
the keying). \textbf{Scope}: single-user assistant memory; multi-writer stores and
adversarial updates are untested.

\section{Conclusion}
Treating agent memory as a self-consistent belief state rather than a document index
makes it robust to the way real knowledge behaves: it changes. \tenet{} stays correct
under knowledge churn where retrieval memory structurally collapses, exceeds the
published single-hop conflict-resolution SOTA at its reader tier on a weaker backbone,
and is the only published-competitive system on MemoryAgentBench whose ingestion and
reads are both LLM-free. The belief-state view also yields time-travel and principled
forgetting for free. We hope knowledge churn becomes a standard axis for evaluating agent
memory.
